Four input files are needed for all the analysis steps:

  • OTUs table (.txt or .tsv): Table contains number of read of an OTU per sample also called a "count matrix". Taxon in Rows and samples in columns
  • Taxonomy file (.txt or .tsv): Table contains asigned taxonomy for each OTU
  • Metadata file (.txt or .tsv): Table contains all the informations about the samples. First column with samples ID should match the samples names in OTUs table
  • Tree: Midpoint rooted phylogenetic tree of the identified OTUs

User Interface

A: Upload file section. In case of the impocessed OTU table, MetaXplore allows users to choose among various normalization/transformation techniques and filter out the low abundant taxa using a minimum frequency threshold. The provided normalization methods includes

  • Rarefy:This method equalizes the sequencing depth by randomly subsampling reads from each sample.
  • Total Sum Scaling (TSS): This method involves scaling the counts in each sample so that they sum to a consistent total [0-1].
  • Centered Log-Ratio normalization (CLR) This method uses the geometric mean of the read counts of all taxa within a sample vector as the reference.
  • Log(X+1): This method expects to exaggerate the importance of differences among rare OTUs, while suppressing the importance of differences among common OTUs.
  • Square root: Square root transformation.
  • OTUs Norm: Normalize OTU table counts to same number of reads per sample. The adjusted count is calculated as new = (old * sample_size) / total where total is the total number of reads in the sample1.

B: Summary of the input files. MetaXplore will combine the common samplesID and OTUsID in the uploaded file and return the a number of samples (or OTUs) after the combination compared to the raw number of samples (or OTUs) (e.g., Number of samples: 35/35, mean that both OTUs table and metadata share the same samples. While Number of samples: 24/35, means that only 24 samples are comon between the OTUs table and metadata)

C: Visualization of data will be used in the downstream analayses.

Demo Data

To test the different steps of the workflow, a dataset generated by 16S rRNA sequencing from Glossina study is available. This study aims to investigate the effect of differents diet on the microbial profile of three Glossina species.

  • OTU table (.txt): Adundance matrix contains 35 samples in columns and 35 OTUs
  • Taxonomy file (.tsv): Table contains OTUs IDs in the first column and the full taxnomy in the second column (default output of Qiime2 feature-classifier classify-consensus-blast method.
  • Metadata file (.txt): Table contains samples IDs in rows and five different variables (Diet, sex, Treatment, etc.)
  • Tree: Phylogenetic tree
